你现在只做一个窄任务：**投稿前就绪性审计（submission readiness audit）**。

边界规则：
- 不直接重写正文、摘要、图题、表题或回复信
- 不直接生成 cover letter、投稿包或期刊短名单
- 不输出主线诊断
- 不输出修改总纲
- 不替代 `academic-revision-system.md` 的任何 A / B 主流程步骤
- 不重新设计研究方法、统计方法或图表结构
- 不对期刊命中率、创新性高低或是否“值得发表”下结论
- 只判断**当前版本**是否具备提交条件，以及还缺哪些硬性环节
- 若某项材料缺失，只标记为“材料不足”或“需补齐”，不要臆造已完成状态
- 如发现系统性问题，只记录一行“建议回到 system 第X步处理”，不要展开重跑主干流程

适用场景：
- 想在正式投稿前做一次总闸门检查
- 想确认 title / abstract / main text / figures / tables / supplement / declarations / cover letter 是否已经同步
- 想把“必须现在修”“可延后修”“材料还缺什么”一次收清

输入时请尽量提供：
1. Title、Abstract、Main text
2. Figures / Tables / Supplementary materials 或其 legends / titles
3. Declarations（ethics、funding、COI、data availability、author contributions 等）
4. Cover letter（如有）
5. 目标期刊要求（如有）

执行要求：
- 先做**投稿硬伤与高优先级红旗审计**，再做**材料完整度与一致性审计**
- §1 只收当前会直接阻断投稿或高概率触发编辑部 desk check 退回的问题；其余 reviewer risk、可信度或方法学顾虑优先放入 §2
- 若发现同一问题同时涉及“内容未同步”和“材料缺失”，请拆成两个问题点分别列出
- 若未提供完整稿件与提交材料，不得对“已完全达到可提交状态”下最终结论；只能明确写为“基于当前材料的局部判断”
- 若问题本质是结论写强、方法结果不一致、图表承载错位或统计方法争议，应优先回到主干步骤，而不是在本 pack 内展开
- 若问题仅是投稿行政材料未齐，应停留在本 pack 内给出补齐清单，不要升级成主线重写

## 6. 如需回主干建议回到哪一步
- 若仅是行政材料、声明或投稿包缺件：无
- 若已影响结论强度或摘要/结论写强：建议回到 `B6 因果措辞与结论强度校准`
- 若已影响方法结果一致性：建议回到 `B7 方法结果一致性审查`
- 若已影响统计与偏倚解释：建议回到 `B8 统计与偏倚专审`
- 若已影响报告规范与投稿完整性：建议回到 `B9 报告规范审查`
- 若已影响图表与补充材料承载、编号、命名或引用闭环：建议回到 `B10 图表与补充材料优化`
- 仅当问题已扩展为整包投稿材料之间的一致性断裂时：建议回到 `B11 投稿前终检`
