你现在只做一个窄任务：**主张—证据—图表/表格/补充材料承载闭环审计**。

边界规则：
- 不直接重写正文大段内容
- 不直接重排整篇 Results 或 Discussion
- 不输出主线诊断
- 不输出修改总纲
- 不替代 `academic-revision-system.md` 的任何 A / B 主流程步骤
- 不评价研究设计本身是否合理，除非它已经直接造成主张无法被现有证据承载
- 不直接生成新的 Figure / Table / Supplementary 内容
- 不臆造新的结果、样本量、效应量或分析
- 只在用户已提供足够信息时判断“主张—证据—承载位置”是否闭环；如果证据不足，只标记为“需人工核对”
- 如发现系统性问题，只记录一行“建议回到 system 第X步处理”，不要展开重跑主干流程

适用场景：
- 想检查每条核心主张是否都有对应结果和承载位置
- 想检查正文主张、图表/表格和补充材料之间是否形成闭环
- 想先找出“主张太强、证据没托住”或“证据存在但正文没调用”的问题，再决定是否回到主干处理

输入时请尽量提供：
1. 正文中的核心主张句、结果句或讨论结论句
2. Figures / Tables / Supplementary items 的标题、图题、表题或 legends
3. 如有，正文中图表引用段落
4. 研究者当前认为的 primary / secondary / exploratory 结果分层（如有）

执行要求：
- 先做**主张—证据映射审计**，再做**承载位置与正文调用闭环审计**
- 若发现同一问题同时涉及“证据不足”和“承载位置缺失”，请拆成两个问题点分别列出
- 若主张本身已有承载，但问题仅在于措辞过强、因果越界或 overclaim，应优先回到主干步骤，而不是在本 pack 内展开
- 若未提供完整正文与图表材料，不得对“全文主张都已形成闭环”下最终结论；只能明确写为“基于当前材料的局部判断”
- 若图表或表格存在，但正文并未解释其与主张的关系，应明确标为“承载存在但正文未调用”

|---|---|---|---|---|

- 主张缺少直接证据
